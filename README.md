# SOTER

> **Сотер** (др.-греч. Σωτήρ — «спаситель», «избавитель»)
> 
> Эпитет (эпиклеса) в древнегреческой мифологии и религии, присваиваемый богам-защитникам, оберегающим от опасности, бедствий или приносящим спасение.

Проект децентрализованной и одноранговой сети и мессенджера на ней, разрабатываемый как проект, цель которого быть независимым от внешней инфраструктуры, быть цензуростойким и невидимым для цензоров. Включает в себя также модель децентрализованной модерации для "демократической" модерации контента внутри сети самой сетью.

## Детали разработки проекта

На данный момент находится в *активной разработке*, пока что проект протестирован на `x86_64 glibc Linux`, имеет в себе Linux-специфичные механизмы. В планах имеется крос-платформенность (Windows, MacOS)

Разработка полностью ведется на языке программирования Си.

## Ход разработки

1. **Базовый этап**
- [x] Создание инструмента пробива NAT
- [x] Создание абстракции для общения через NAT 2ум пирам (клиентам)
- [x] Полная реализация протокола общения через NAT по UDP с подтверждением пакетов, их целостностью

2. **Криптография**
- [ ] Наложение на протокол шифрования в виде элиптических кривых и шифров AES-256 / chacha20
- [ ] Совмещение с TLS1.2/1.3 для обфускации

3. **Анти-цензурные механизмы**
- [ ] Внедрение разработок X-Ray core (XHTTP)

4. **Создание самого мессенджера**
- [ ] Удобное API с переходом на HTTP, которое даст возможность коммьюнити создавать свои версии клиентов мессенджера

5. **Внедрние модерационного механизма**
- [ ] Модерация внутри мессенджера

## Как собрать проект

1. Для доп. установки и настройки рабочей директории перед компиляцией:
   
```
./SetupScript
```

2. Для сборки `main.c` в `code/`

```
make
```

3. Запуск скомпилированного приложения

```
make run
```
